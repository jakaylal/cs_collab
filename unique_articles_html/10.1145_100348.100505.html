          Abstract

          One reason that Pascal is considered an appropriate introductory programming language for computer science majors is that Pascal permits the creation of modular programs by using logically independent procedures. However this advantage of Pascal is utilized only if students learn to manage parameter passing well enough to harness Pascal's power. In particular, students should learn to use the stylistic conventions accepted by the professional programming community and to understand the reasons for these conventions. This study examined student knowledge and beliefs relating to parameter passing in Pascal. During interviews, students were asked to predict which programs from a collection of similar programs would work according to specification and which would not work due to variable access errors or parameter passing errors. Also during interviews, students were asked to rate program versions on several stylistic criteria, including ease of comprehension, ease of debugging, and ease of modification. Programming experts were asked to rate the same program versions on the same criteria. This paper will discuss these inter-views. In particular students' views of the larger programming process will be discussed.
